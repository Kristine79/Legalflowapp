# LegalFlow — Product Requirements Document

## 1. Продукт в двух словах

LegalFlow — AI SaaS для управления юридической практикой в России. Продукт помогает юристам и фирмам принимать обращения, вести дела и задачи, работать с документами и получать структурированные рекомендации от нейросети.

## 2. Целевая аудитория

- **Юристы-одиночки** — хотят автоматизировать приём клиентов и не терять дела.
- **Малые и средние юридические фирмы** — нуждаются в общей базе клиентов, задачах и ролевой модели.
- **Ассистенты и параюристы** — работают с документами и задачами под руководством юристов.
- **Владельцы фирм** — следят за воронкой и загрузкой команды через аналитику.

## 3. Пользовательские сценарии

### 3.1 Приём обращения
1. Потенциальный клиент звонит или пишет.
2. Юрист вносит ФИО, контакты и описание проблемы.
3. AI анализирует обращение: категория, приоритет, риски, нужные документы, следующий шаг.
4. Создаётся карточка клиента со статусом «Новое обращение».
5. Telegram-бот уведомляет ответственного сотрудника.

### 3.2 Ведение дела
1. Клиент переходит в статус «Дело в работе».
2. Юрист создаёт задачи с дедлайнами и назначает исполнителей.
3. Загружаются документы; AI помогает проанализировать договор.
4. Все действия фиксируются в истории активности.

### 3.3 Контроль и аналитика
1. Владелец/админ открывает дашборд.
2. Видит KPI: всего клиентов, новых сегодня, в работе, закрыто.
3. Смотрит тренд по дням, распределение по статусам и воронку.
4. AI-блок даёт рекомендации по работе с активными клиентами.

## 4. Функциональные требования

### 4.1 Авторизация и доступ
- Регистрация и вход через email/пароль и Google OAuth (Clerk).
- Пять ролей: `owner > admin > lawyer > assistant > client`.
- Ролевая фильтрация применяется на уровне SQL-запросов.
- Онбординг при первом входе: имя, название фирмы, Telegram chat ID.
- Русская локализация всех экранов Clerk (deep-merge `ruRU` + custom overrides).

### 4.2 Клиенты
- Список клиентов с фильтрами и поиском.
- Создание/редактирование карточки: ФИО, телефон, email, описание, статус.
- Статусы: `new`, `in-progress`, `waiting`, `closed`.
- AI-анализ первичного обращения (summary, category, priority, risks, questions, documents, nextAction).
- История активности по клиенту.
- Удаление клиента с записью в аудит.

### 4.3 Дела
- Дело привязано к клиенту.
- Статусы дела: `new-request`, `consultation`, `documents`, `court`, `closed`.
- Назначение юриста и ассистента.
- Связь с документами и задачами.

### 4.4 Задачи
- Создание, редактирование, удаление задач.
- Поля: название, описание, статус (`pending`/`done`), приоритет, дедлайн, назначенный исполнитель, связь с делом/клиентом.
- Быстрое переключение статуса.
- Фильтрация по статусу и приоритету.
- Фронтенд работает через сгенерированные React Query хуки (`use-tasks.ts`).

### 4.5 Документы
- Загрузка файлов: PDF, DOCX, DOC, ODT, TXT, MD, RTF (до 10 МБ).
- Извлечение текста при загрузке и сохранение в БД.
- AI-анализ договора: summary, риски, спорные пункты.
- Вопросы к документу (Q&A).
- Скачивание исходного файла.

### 4.6 Уведомления
- In-app список уведомлений.
- Telegram-уведомления при создании клиента и важных событиях.
- Тестовая отправка уведомления из настроек.

### 4.7 Аналитика
- Эндпоинт `GET /analytics` возвращает `DashboardStats`.
- KPI-карточки: `totalClients`, `newToday`, `inProgress`, `closed`.
- Недельный тренд новых клиентов.
- Распределение по статусам (круговая диаграмма).
- Воронка из 5 этапов (`FunnelChart`).
- AI-инсайты на дашборде.
- Демо-страница `/demo` показывает аналитику без авторизации через `overrideStats`.

### 4.8 Лендинг и публичные страницы
- Hero с CTA на регистрацию и демо.
- Сетка ключевых фич.
- Демо-видео (GIF из реальных скриншотов `/demo`).
- Блок доверия: шифрование, 152-ФЗ, безопасность.
- Отзывы и социальное доказательство.
- Тарифы (Freemium / Pro / Enterprise).
- FAQ-раздел.
- Footer со ссылками на `/about`, `/pricing`, `/faq`.
- Общая навигация `PublicHeader` для `/about`, `/pricing`, `/faq`.

### 4.9 Настройки
- Профиль пользователя: имя, инициалы, фирма, email.
- Telegram chat ID и toggle уведомлений.
- Управление пользователями и ролями (owner/admin).
- Сброс онбординга.
- Синхронизация профиля с БД через `POST /users/sync`.

## 5. Нефункциональные требования

- **Производительность:** время ответа API < 500 мс для типичных запросов, загрузка страницы < 2 с.
- **Масштабируемость:** stateless API, файлы хранятся на локальной ФС (план — объектное хранилище).
- **Безопасность:** аутентификация через Clerk, RBAC на уровне SQL, валидация входных данных Zod, аудит действий.
- **Надёжность:** graceful degradation AI-фич (503 → локальная эвристика на фронте).
- **Локализация:** интерфейс на русском, i18n-объект подготовлен для английского.
- **Типобезопасность:** генерация типов из OpenAPI, `tsc --noEmit` без ошибок.

## 6. Технологический стек

- **Монорепозиторий:** pnpm workspaces
- **Frontend:** React 19, Vite, Tailwind CSS v4, shadcn/ui, Wouter, Framer Motion, Recharts
- **Backend:** Express 5, TypeScript, pino-http
- **Auth:** Clerk (`@clerk/react`, `@clerk/express`, `@clerk/localizations`)
- **Database:** PostgreSQL + Drizzle ORM
- **Validation:** Zod v3 + drizzle-zod
- **API codegen:** Orval v8
- **AI:** OpenAI-совместимый провайдер (Replit AI proxy / OpenRouter / OpenAI)
- **Telegram:** Bot API
- **Storage:** локальная ФС (`artifacts/api-server/uploads/`)

## 7. Модель данных (кратко)

- `users` — профили и настройки пользователей.
- `clients` — клиенты и их AI-анализ.
- `cases` — дела, статусы, назначения.
- `tasks` — задачи, статусы, дедлайны, приоритеты, назначения.
- `documents` — загруженные файлы и извлечённый текст.
- `activities` — история событий по клиентам/делам.
- `notifications` — in-app уведомления.
- `teamMembers` — роли и связи внутри фирмы.
- `auditLogs` — аудит действий.

## 8. UI/UX принципы

- Минималистичный LegalTech стиль: чистая типографика, приглушённые цвета, акцентный синий.
- Профессиональность и доверие: структура, иконки, консистентные отступы.
- Рабочий инструмент, а не демо: плотные, но не перегруженные экраны; быстрые фильтры и статусы.
- Адаптивность: корректная работа на десктопе и планшете; мобильная версия — упрощённая, но функциональная.
- Empty states: понятные сообщения и CTA при отсутствии данных.
- Состояния загрузки: скелетоны и спиннеры в ключевых местах, без блокировки всего интерфейса.

## 9. API и интеграции

- **OpenAPI:** `lib/api-spec/openapi.yaml` — источник истины.
- **Роуты:** `ai`, `analytics`, `users`, `clients`, `cases`, `activities`, `tasks`, `notifications`, `calendar`, `telegram`, `health`.
- **Clerk proxy:** production-only middleware для custom domains.
- **AI waterfall:** Replit AI proxy → OpenRouter → OpenAI.
- **Telegram:** уведомления через Bot API по `telegramChatId` пользователя.

## 10. Безопасность и соответствие

- Шифрование данных в пути (HTTPS) и покоя (PostgreSQL на Replit).
- Соответствие 152-ФЗ «О персональных данных» на уровне обещаний продукта (шифрование, ограничение доступа, аудит).
- RBAC с разграничением доступа к клиентам и делам.
- Аудит действий в `auditLogs`.

## 11. Метрики успеха (North Star)

- Время от первичного обращения до создания карточки клиента < 2 минут.
- Доля задач, закрытых в срок.
- Рост конверсии воронки (новое обращение → дело в работе → закрыто).
- Удовлетворённость пользователей ( retention через 7 и 30 дней ).
