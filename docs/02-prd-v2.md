# PRD v2.0 — Product Requirements Document

## Общая информация

**Product Vision:**  
Сделать LegalFlow стандартным инструментом для юристов, который позволяет вести дела, анализировать документы и управлять сроками быстрее, чем в комбинации Word / Excel / Telegram / Email.

**Текущий стек:** React 19 + Vite + Node.js + Express 5 + Drizzle ORM + PostgreSQL + Clerk + OpenAI/OpenRouter + Telegram Bot API.

## Целевая аудитория

- Юристы малого и среднего бизнеса (B2B)
- Индивидуальные юристы и адвокаты
- Legal-отделы крупных компаний (в качестве пилота)

## User Personas

| Персона | Роль | Потребности |
|---|---|---|
| **Анна, корпоративный юрист** | Практикующий юрист | Скорость подготовки документов, защита от ошибок, быстрый доступ к шаблонам |
| **Павел, адвокат** | Самозанятый юрист | Единое хранилище дел, напоминания о сроках, история клиентов |
| **Екатерина, руководитель отдела** | Управленец | Аналитика по делам, контроль загрузки команды, прозрачность процессов |

## Функциональные требования (MVP)

### Модуль 1 — Библиотека документов и шаблонов
- Категоризация документов (договоры, претензии, доверенности)
- Загрузка и хранение файлов (.docx, .pdf, .txt)
- Просмотр, скачивание, удаление
- AI-анализ содержимого (риски, ключевые пункты, сроки)

### Модуль 2 — Заполнение и автозаполнение
- Ручной ввод данных клиента и дела
- AI-генерация шаблонов на основе кейса
- Предпросмотр сгенерированного документа перед сохранением

### Модуль 3 — Согласование (базовое)
- Назначение ответственных на задачи
- Комментарии и статусы задач
- Версионность документов (на уровне сущностей в БД)

### Модуль 4 — Управление сроками
- Календарь событий и дедлайнов
- Автоматические напоминания через Telegram
- Визуальное выделение просроченных задач

### Модуль 5 — AI-инструменты
- Анализ интейка клиента (категория дела, следующие шаги)
- Генерация документов по шаблону и кейсу
- Извлечение сроков и рекомендаций по задачам
- Генерация summary по делу
- Проверка конфликтов интересов с существующими клиентами

## Нефункциональные требования

- **Производительность:** генерация AI-ответа — не более 10 секунд; загрузка страницы CRM — < 2 секунд
- **Безопасность:** шифрование данных в транзите (TLS), аутентификация через Clerk, ролевая модель RBAC
- **Доступность:** 99.5% uptime (целевой)
- **Локализация:** поддержка RU/EN
- **Масштабируемость:** монорепозиторий на pnpm, развёртывание через Replit workflows

## Приоритеты (MoSCoW)

- **Must have:**
  - Регистрация / вход через Clerk
  - Клиенты, дела, задачи, календарь
  - Загрузка и хранение документов
  - AI-анализ документов и генерация шаблонов
  - Telegram-уведомления
- **Should have:**
  - Автозаполнение реквизитов из внешних источников
  - Расширенная аналитика и отчёты
- **Could have:**
  - Интеграция с внешними CRM (AmoCRM, 1С)
  - Мобильное приложение
- **Won't have (v2):**
  - Продвинутые AI-рекомендации по условиям договоров
  - Интеграция с ЭЦП и Госключ

## Критерии приёмки (пример: «Генерация AI-анализа документа»)

1. Пользователь открывает документ и нажимает «Анализировать».
2. Система извлекает текст из .docx/.pdf.
3. AI возвращает структурированный ответ: риски, обязательства, ключевые даты.
4. Результат отображается в интерфейсе и сохраняется в истории документа.
5. Время отклика ≤ 10 секунд при стабильном интернете.
