# План MVP — LegalFlow

## Цель MVP

Запустить рабочий цикл «клиент → дело → документ → задача → уведомление» для 10 пилотных пользователей и получить обратную связь на реальных кейсах.

## Состав фич (Must have)

1. **Аутентификация и роли**
   - Регистрация / вход через Clerk
   - Роли: owner, attorney, paralegal
   - Синхронизация пользователя в БД при первом входе

2. **Клиенты и дела**
   - Создание, редактирование, архивирование клиентов
   - Создание дел с привязкой к клиенту, статусом, категорией, дедлайном

3. **Библиотека документов**
   - Загрузка файлов (.docx, .pdf, .txt)
   - Просмотр, скачивание, удаление
   - Связь документа с делом

4. **AI-анализ документов**
   - Извлечение текста из PDF и DOCX
   - AI-анализ рисков, обязательств, сроков
   - Сохранение результата в истории документа

5. **AI-генерация шаблонов**
   - Генерация текста документа по описанию кейса
   - Возможность сохранить результат как новый документ

6. **Задачи**
   - Создание задач с дедлайном, статусом, ответственным
   - Kanban-визуализация по статусам
   - Поиск и фильтрация задач

7. **Календарь**
   - Создание событий (судебные заседания, встречи, дедлайны)
   - Привязка событий к делам

8. **Уведомления в Telegram**
   - Подключение Telegram-бота в настройках
   - Автоматические напоминания о задачах и дедлайнах

9. **Дашборд и аналитика**
   - Воронка дел
   - График активности
   - Быстрые действия и статистика

10. **Локализация**
    - Русский и английский интерфейс
    - Локализация Clerk-форм для русских пользователей

## Оценка трудозатрат (пример)

| Роль | Часы |
|---|---|
| Backend (Node.js/Express/Drizzle) | 180 |
| Frontend (React/Vite/Tailwind) | 160 |
| AI-интеграция и парсинг документов | 60 |
| QA (ручное + smoke-автоматизация) | 70 |
| DevOps / Replit workflows | 40 |
| **Итого** | **~510 часов** |

При команде из 3–4 человек MVP занимает примерно 3 календарных месяца.

## Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| Долгие ответы от OpenAI/OpenRouter | Средняя | Среднее | Таймауты, индикаторы загрузки, кэширование результатов |
| Нестабильность парсинга PDF/DOCX | Средняя | Высокое | Использовать проверенные библиотеки, fallback на текстовые файлы |
| Пользователи не подключают Telegram | Высокая | Среднее | Альтернативный канал: email-уведомления в следующей итерации |
| Пропуск ролей / RBAC | Низкая | Критическое | Покрытие middleware тестами, аудит действий |
| Рост объёма файлов | Средняя | Среднее | Ограничение размера файла, мониторинг `UPLOADS_DIR`, миграция на S3 в будущем |

## Критерии готовности MVP

- [ ] Все must-have фичи реализованы и протестированы
- [ ] Smoke-тесты проходят без ошибок
- [ ] 10 пилотных пользователей зарегистрированы
- [ ] Среднее время AI-анализа ≤ 10 секунд
- [ ] Нет критических багов в аутентификации и хранении документов
- [ ] Документация (README + PRD) актуальна
