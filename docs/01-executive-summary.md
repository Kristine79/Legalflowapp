# Executive Summary — LegalFlow

**Название проекта:** LegalFlow  
**Версия:** 1.0  
**Дата:** 13 июля 2026 г.

## Проблема

Юридические отделы и малые юрфирмы в России тратят значительную часть рабочего времени на рутинную подготовку документов, ручной контроль сроков и согласование версий. Отсутствие единой платформы приводит к ошибкам, пропущенным дедлайнам и потере клиентов.

## Решение

LegalFlow — облачная платформа для управления юридической практикой с элементами искусственного интеллекта. На текущем этапе реализованы:

- **CRM для юрфирмы:** клиенты, дела, задачи, календарь, документы
- **AI-ассистент:** анализ документов, генерация шаблонов, извлечение сроков, проверка конфликтов интересов
- **Уведомления:** интеграция с Telegram Bot API для напоминаний
- **Аналитика:** дашборд с воронкой дел и активностью
- **Аутентификация и роли:** Clerk (owner, attorney, paralegal)

## Бизнес-ценность

- Сокращение времени на первичный анализ документов
- Централизованное хранилище дел, документов и задач
- Снижение риска пропущенных сроков за счёт уведомлений
- Быстрый доступ к истории клиентов и предыдущим документам

## Ключевые метрики (MVP)

- 10+ активных пользователей в первый месяц
- 100+ успешно созданных документов / задач
- Средний NPS ≥ 40
- Среднее время генерации AI-анализа < 10 секунд

## Архитектура

- **Frontend:** React 19 + TypeScript + Vite 6 + Tailwind CSS 4
- **Backend:** Node.js + Express 5 + TypeScript (esbuild bundle)
- **ORM:** Drizzle ORM
- **База данных:** PostgreSQL
- **Аутентификация:** Clerk
- **AI:** OpenAI / OpenRouter API
- **Уведомления:** Telegram Bot API
- **Сборка:** pnpm workspaces, Replit workflows
