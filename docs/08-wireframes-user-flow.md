# Wireframes / User Flow — LegalFlow

## Экран 1 — Главный / Дашборд

**Элементы интерфейса:**
- Верхняя панель: логотип LegalFlow, глобальная строка поиска, аватар пользователя, переключатель языка
- Боковая панель: Дашборд, Клиенты, Дела, Документы, Задачи, Календарь, AI-инструменты, Настройки
- Центральная зона:
  - Блок «Аналитика» — карточки: активные дела, задачи на сегодня, клиенты, документы
  - Воронка дел (funnel chart)
  - График активности за последние 7 дней
  - Блок «Последние документы» — список с датой и статусом
- Кнопка «Новый документ» / «Новая задача»

## Экран 2 — Выбор шаблона / AI-генерация

**Элементы интерфейса:**
- Раздел AI-инструменты
- Поле ввода описания дела
- Выбор типа документа (аренда, подряд, услуги, NDA, купля-продажа)
- Кнопка «Сгенерировать»
- Область предпросмотра сгенерированного текста
- Кнопки «Скопировать», «Сохранить как документ», «Начать заново»

## User Flow: «Создать и проанализировать договор аренды»

1. Пользователь нажимает «Новый документ» на дашборде.
2. Переходит в раздел «Документы» и загружает файл `.docx` или `.pdf`.
3. Система сохраняет файл и отображает его в списке.
4. Пользователь открывает документ и нажимает «Анализировать».
5. AI извлекает текст и возвращает структурированный анализ:
   - Риски
   - Обязательства сторон
   - Ключевые даты и сроки
   - Рекомендации
6. Пользователь может задать уточняющий вопрос по документу (Q&A).
7. Результат сохраняется в истории документа.
8. Пользователь создаёт задачу «Согласовать договор аренды» с дедлайном.
9. При приближении дедлайна приходит уведомление в Telegram.

## Экран 3 — Согласование / Задачи

**Элементы интерфейса:**
- Фильтры по статусу: «Все», «В работе», «На проверке», «Завершено», «Просрочено»
- Глобальный поиск по названию и описанию задачи
- Kanban-доска или список задач
- Внутри задачи:
  - Заголовок и описание
  - Дедлайн и статус
  - Ответственный
  - Связанное дело / документ
  - Кнопки «Редактировать», «Завершить», «Удалить»
- Просроченные задачи выделены красным

## Экран 4 — Документ

**Элементы интерфейса:**
- Название документа и дата загрузки
- Ссылка на связанное дело и клиента
- Кнопки: «Скачать», «Удалить», «Анализировать»
- Вкладка «Анализ» — результат AI-анализа
- Вкладка «Вопросы» — история Q&A по документу
- Вкладка «История» — активность по документу

## Экран 5 — Настройки / Telegram

**Элементы интерфейса:**
- Раздел «Интеграции»
- Инструкция по подключению Telegram-бота (3 шага)
- Поле для ввода chatId
- Кнопка «Сохранить»
- Статус подключения: «Подключено» / «Не подключено»
- Переключатель языка (RU/EN) и темы (светлая/тёмная)

## Экран 6 — Календарь

**Элементы интерфейса:**
- Месячный / недельный / дневной вид
- Список событий с цветовой маркировкой по типу (судебное заседание, встреча, дедлайн)
- Кнопка «Новое событие»
- Форма события: название, дата/время, тип, место, связанное дело, описание

## Экран 7 — Профиль / Команда

**Элементы интерфейса:**
- Данные пользователя (из Clerk)
- Роль в фирме
- Список участников команды (для owner)
- Возможность назначить роль (attorney / paralegal)
- Условия и тарифы (заглушка для future pricing)

## Принципы навигации

- Все основные действия доступны за 2 клика с дашборда.
- Глобальный поиск доступен на каждой странице CRM.
- Формы валидации сразу показывают ошибки.
- AI-функции обозначены отдельной иконкой/статусом, чтобы пользователь понимал, когда используется AI.
